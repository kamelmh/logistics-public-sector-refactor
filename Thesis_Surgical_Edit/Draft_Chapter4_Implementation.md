# الفصل الرابع: التنفيذ العملي والبرمجة (Implementation)

## المبحث الأول: بناء وحدات المعالجة المركزية (VBA Core Engine)

### 1. المنهجية المعتمدة
تم تطوير محرك المعالجة باستخدام لغة VBA (Visual Basic for Applications) بشكل متكامل، وذلك لضمان استقرار النظام وتجنب الاعتماد على برمجيات خارجية. تم تقسيم المحرك إلى وحدات وظيفية مستقلة (Modular Design) لضمان سهولة الصيانة والتطوير.

### 2. هيكلية الوحدات البرمجية
يعتمد النظام على توزيع المهام بين الوحدات التالية:
- **`mod_StockEngine.bas`**: يتولى كافة العمليات الحسابية المعقدة (حساب EOQ، تحديد ROP، وتصنيف ABC-XYZ).
- **`mod_SyncBridge.bas`**: يعمل كمنسق للبيانات، حيث يضمن تحديث جميع الجداول المرتبطة فور تسجيل أي حركة مخزنية.
- **`mod_Database.bas`**: يتولى عمليات الكتابة والقراءة الآمنة من جداول Excel.

### 3. ضمان سلامة البيانات (Data Integrity)
تم تطبيق بروتوكول "الكتابة الآمنة" لضمان عدم فقدان البيانات:
- **الحماية البرمجية**: يتم إلغاء حماية الجداول لحظياً أثناء الكتابة وإعادة تفعيلها فوراً.
- **التحديث الذري**: يتم تنفيذ عمليات التحديث في كتلة واحدة، وفي حال حدوث خطأ، يتم استدعاء إجراء التراجع (Rollback) لحذف السجلات غير المكتملة.

## المبحث الثاني: تطوير واجهات الإدخال والرقابة

### 1. نظام الحراسة والتحقق (Validation System)
تم دمج منطق التحقق من الرصيد مباشرة في واجهة الإدخال لمنع حدوث أخطاء في المخزون:

```vba
' mod_StockEntry_Logic.bas
Public Sub ValidateStock()
    If mvtType = "OUT" Then
        If qty > currentStock Then
            MsgBox "Stock insuffisant!", vbCritical
            Exit Sub
        End If
    End If
    ' تحديث ورقة البيانات مباشرة عبر mod_StockEngine
    Call mod_StockEngine.UpdateArticleStockBalance(artCode, mvtType, qty)
End Sub
```

### 2. حساب التكلفة المتوسطة المرجحة (CMUP)
يتم إعادة احتساب CMUP آلياً بعد كل عملية إدخال لضمان دقة تقييم المخزون مالياً:
```
CMUP_nouveau = [(Total_In_Value + Current_Stock_Value)] / (Total_In_Qty + Current_Stock_Qty)
```

## المبحث الثالث: تكامل البيانات وتدفق المعلومات (Local Data Flow)

### 1. بنية البيانات المحلية (Excel-Based Storage)
تم اعتماد جداول Excel المنظمة كقاعدة بيانات مركزية لضمان:
- **الاستقلالية التقنية**: عدم الارتباط بقواعد بيانات خارجية أو خوادم.
- **السرعة**: الوصول الفوري للبيانات دون الحاجة لعمليات استعلام معقدة.

### 2. بروتوكول تدفق البيانات (Data Flow)
1. **الإدخال**: المستخدم يسجل الحركة عبر واجهة `frmStockEntry`.
2. **المعالجة**: تقوم VBA بحساب تأثير الحركة على الرصيد وتحديث قيم CMUP.
3. **التسجيل**: يتم تدوين الحركة في سجل `MOUVEMENTS` وتحديث الرصيد النهائي في سجل `ARTICLES`.
4. **التنبيه**: يقوم النظام بإصدار تنبيه فوري إذا وصل الرصيد إلى نقطة الطلب (ROP).

### 3. التوافق الإداري واللغوي
تم تحسين الواجهات لتدعم اللغة العربية بشكل كامل، مع استخدام ترميز متوافق يضمن ظهور النصوص بشكل صحيح في كافة تقارير Excel الموجهة للإدارة.
