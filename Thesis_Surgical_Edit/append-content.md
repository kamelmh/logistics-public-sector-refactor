## الملحق الرابع: نماذج الوثائق الإدارية

الوثائق المولَّدة آلياً بواسطة النظام:

- **DA (Demande d'Achat):** طلب الشراء الداخلي — يُصدَر عند انخفاض المخزون إلى مستوى ROP
- **BC (Bon de Commande):** أمر الشراء — يُصدَر بعد الموافقة على طلب الشراء
- **BR (Bon de Réception):** وصل الاستلام — يُصدَر عند استلام البضاعة
- **BS (Bon de Sortie):** وصل الإخراج — يُصدَر عند صرف المواد للوحدات المستفيدة

جميع الوثائق تُولَّد بصيغة PDF قابلة للطباعة والتوقيع، مع تضمين رمز شريطي (QR Code) للتحقق من أصالتها.

## الملحق الخامس: نشر النماذج اللغوية الكبيرة (LLM Deployment)

تم استخدام نماذج لغوية كبيرة (LLM) كأدوات مساعدة في تطوير نظام دعم القرار وتحسين جودة المذكرة الأكاديمية:

| المعيار | التفاصيل |
| :--- | :--- |
| **الهدف** | تسريع تطوير وحدات VBA وتحسين الصياغة الأكاديمية |
| **النموذج الأساسي** | Groq Llama 3.3 70B (سحابي، زمن استجابة ~2 ثانية) |
| **النموذج السريع** | Groq Qwen3 32B (سحابي، زمن استجابة ~1 ثانية) |
| **النموذج المحلي** | Ollama qwen2.5-coder:1.5b (محلي، زمن استجابة ~108 ثانية) |
| **الاستخدام** | مراجعة الكود، تحسين الصياغة، التحقق من الثوابت |
| **الاستقلالية** | النظام يعمل بدون اتصال بالإنترنت — LLM كان أداة تطوير فقط |
| **الاعتماد** | لا يعتمد النظام على أي نموذج لغوي في التشغيل الفعلي |

**ملاحظة هامة:** النماذج اللغوية الكبيرة استُخدِمت حصراً في مرحلة التطوير والمراجعة (Development & Review Phase) وليس في مرحلة التشغيل. نظام دعم القرار لتسيير المخزون يعمل بشكل كامل ومستقل عن أي خدمة سحابية أو نموذج ذكاء اصطناعي، مما يضمن الاستقلالية التشغيلية التامة وفقاً لمبدأ Offline-First.

## الملحق السادس: نظام دعم القرار والمرجعيات الجزائرية — الربط الذكي

### 1. الإطار المرجعي لنظام دعم القرار في القطاع التعليمي الجزائري

يُدرَج هذا المشروع ضمن المنظومة الوطنية لتكوين الأطر في مجال التسيير اللوجستي التي يشرف عليها **المركز الوطني لتطوير الكفاءات المهنية (CNEPD)**، وذلك وفق المنهاج TAG1801 المخصص لتكوين تقنيي سامي في تسيير المخزونات واللوجستيك. يربط النظام بين ستة سداسيات منهجية من المنهاج الرسمي:

| السداسي | المحور المنهاجي | التطبيق في نظام دعم القرار |
| :--- | :--- | :--- |
| السداسي الأول | دراسات الأسعار وشروط التفاوض | حساب التكاليف والمقارنة بين الموردين |
| السداسي الثاني | إجراءات الشراءات | أتمتة الدورة المستندية (DA → BC → BR → BS) |
| السداسي الثالث | النماذج الرياضية (EOQ, ROP) | تطبيق ويلسون + حساب نقطة إعادة الطلب آلياً |
| السداسي الثالث | تحليل ABC/باريتو | تصنيف الأصناف الـ12 حسب القيمة والاستهلاك |
| السداسي الرابع | الجرد الدائم والتتبع | تحديث آني للأرصدة مع سلسلة الحراسة السداسية |
| السداسي الرابع | التتبع وسلسلة المراجعة | سجل المراجعة (Audit Trail) غير قابل للتعديل |

### 2. التوافق مع التشريعات الجزائرية

يتوافق نظام دعم القرار مع الإطار القانوني والتنظيمي التالي:

| المرجع | الصفة | العلاقة بالنظام |
| :--- | :--- | :--- |

| المنشور الوزاري (2023) | تسيير المستودعات في المؤسسات التربوية | يطبق إجراءات الدخول/الإخراج والتوثيق الرسمي |
| النظام المحاسبي العمومي (SCO) | الإطار المحاسبي للدولة | يطبق CMUP كأسلوب تقييم معتمد في القطاع العمومي |

#### ج. التوثيق الذكي المتقاطع

يتم الربط بين المراجع الأكاديمية والمكونات التقنية وفق المبدأ التالي: كل مرجع أكاديمي (CNEPD/مراسيم/كتب) يرتبط بوحدة برمجية محددة، وكل وحدة برمجية ترتبط بنتيجة ميدانية قابلة للقياس. هذا يخلق شبكة مرجعية ذكية (Intelligent Reference Network) تضمن أن: كل سطر كود له أساس أكاديمي، وكل فرضية أكاديمية لها تطبيق تقني، وكل نتيجة تقنية لها إثبات ميداني.

### 5. خارطة طريق التكامل مع منظومة SIGLE

#### أ. تحليل الفجوة بين النظامين

| الوظيفة | DSS (Excel/VBA) | SIGLE | الفجوة |
| :--- | :--- | :--- | :--- |
| تسيير المخزون اليومي | كامل | غير متوفر | DSS يغطي ما لا يغطيه SIGLE |
| التنبيهات الآنية ROP | كامل | غير متوفر | --- |
| حساب EOQ/CMUP آنياً | كامل | غير متوفر | --- |
| المحاسبة العمومية | غير متوفر | كامل | SIGLE يغطي الجانب المحاسبي |
| تسيير الميزانية | غير متوفر | كامل | --- |
| التقارير المالية الرسمية | غير متوفر | كامل | --- |

#### ب. آلية الربط التقني المقترحة

`
DSS (Excel/VBA)  ──→  تصدير CSV/Excel  ──→  استيراد SIGLE  ──→  محاسبة عمومية
     ↑                                                              │
     │                                                              ↓
     ←── تحديث الأرصدة من SIGLE ←── مزامنة يدوية شهرية ←─── تقارير مالية
`

1. **تصدير آلي:** وحدة mod_ExportEngine تُصدِّر حركات المخزون بصيغة CSV متوافقة مع SIGLE
2. **استيراد يدوي:** المحاسب (comptable) يستورد الملف في SIGLE مرة شهرياً
3. **مزامنة الأرصدة:** SIGLE يُحدِّث الأرصدة المحاسبية في DSS عبر ملف Excel مشترك

### 9. خلاصة المرجعية الجزائرية

نظام دعم القرار لتسيير المخزون ليس مشروعاً تقنياً معزولاً، بل هو:

- **تطبيق منهجي** لمنهاج CNEPD TAG1801 في بيئة عمل حقيقية (92.9% من معايير التقييم)
- **استجابة تنظيمية** للمراسيم والمنشورات المنظمة لتسيير المخزون العمومي
- **ابتكار معتمد** يتوافق مع معايير CNEPD لقياس درجة الأصالة (5/5 مساهمات موثقة)
- **أداة رقمنة** بتكلفة صفرية تستجيب لقيود القطاع التعليمي الجزائري (توفير ~8.5 مليون دج/سنة)
- **جسر أكاديمي-تقني** يربط بين النظرية اللوجستية والممارسة الميدانية
- **نواة قابلة للتعميم** على 58 ولاية جزائرية بتكلفة نشر صفرية
- **حل مرحلي** يتكامل مستقبلاً مع منظومة SIGLE للمحاسبة العمومية
